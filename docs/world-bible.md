# 世界观圣经

## 核心概念

变量城是一座被公共规则引擎托管的现代都市。白天，它像普通城市一样运转：通勤、打卡、会议、外卖、便利店和深夜办公室都很寻常；到了夜里，人们的情绪、承诺、记忆和数字习惯会溢出成“异常数据”。

异常数据不是单纯的错误。它们像城市的余温，既会造成混乱，也能让生活变得不那么机械。夜班调试员的工作不是把所有异常清零，而是在秩序崩溃前找到新的平衡。

城市底层存在多个抽象数据层，例如情绪层、承诺层、摸鱼层、美食层、时间层、动物层与结构层。异常数据是现实场景与这些数据层互相渗透时留下的接口。

公共规则引擎追求稳定、可预测和绝对一致。它会派出“白箱巡检员”回收异常变量。巡检员原本是维护机制，但当规则引擎过度收紧时，也可能把有价值的差异一起擦掉。

## 抽象数据层

- `情绪层`：压力、开心、焦虑会生成可见实体，例如 emo 毛球。
- `承诺层`：承诺、KPI、空头计划会变成带重量的实体或护盾。
- `摸鱼层`：休息与逃避会变成临时避难所，但停留太久会失去行动力。
- `美食层`：食物会拥有自己的路径和脾气，也可能生成补偿效果。
- `时间层`：可短暂回滚、暂停或跳帧，但会产生时间债。
- `动物层`：城市动物拥有自己的异常能力和路线规则。
- `结构层`：整数、浮点数、数组、栈、队列、哈希表等概念会变成战斗属性与房间规则。

## 世界规则

1. 少数人能看见并调试异常数据。这种能力通常来自长期接触规则引擎、持有调试信物，或在高压场景中被异常数据“选中”。
2. 每个异常都有作用和反噬。修复异常会获得 `bug点数`，但过度改写会提升系统关注度。
3. 武器和能力来自日常办公物品、调试工具和数据结构概念。它们既是战斗构筑，也是轻量科普入口。
4. 整数、浮点数、数组、栈、队列等概念可作为属性克制。玩家不需要先懂编程，只要通过玩法感受到“数得清”“会误差”“排成一列”“后进先出”“先来先处理”等直觉。
5. 白箱巡检员会回收异常变量。前期它是高压追击者，后续可通过剧情揭示它也在学习判断“有害异常”和“有价值差异”。
6. 规则引擎会周期性升级。升级失败会制造新的异常日，例如季节错位、房间循环、时间卡帧或数据结构失衡。
7. 人类、异常实体、城市动物和巡检机制都可以协作或对抗。最终目标是让城市规则与自由变量共存。

## 主要角色

### 安渡

- 年龄：24
- 定位：主角，夜班调试员
- 性格：前期只想准点下班、怕麻烦；后期愿意承担责任。擅长临场反应，对异常实体有同理心。
- 目标：查清变量城夜间异常增多的原因，保护城市里那些“不该被清零的差异”。
- 禁忌：不利用异常伤害普通人，不为了效率抹掉有价值的个体差异。
- 口头禅：摸鱼可以，变量不能乱飞。
- 关系：乔柚是搭档兼知己；老梁是前主管兼线索提供者；白箱巡检员前期敌对、后期可能成为盟友。

### 乔柚

- 年龄：23
- 定位：搭档，异常翻译员
- 性格：活泼、细心，有点迷糊；能听懂异常实体的“数据噪声”，擅长降低反噬。
- 目标：帮助安渡稳定变量城，收集异常实体的真实诉求。
- 禁忌：不随意伤害无害异常，不把所有不同都归类为错误。
- 口头禅：别急，我先听听它到底在报什么错。
- 关系：安渡是搭档兼知己；老梁是可靠但嘴硬的前辈；白箱巡检员是需要重新校准的机制。

### 老梁

- 年龄：45
- 定位：配角，前主管，隐藏的规则引擎维护者
- 性格：嘴上现实、爱画流程图，实则重情义。熟悉城市旧版规则，懂得用生活经验解释抽象概念。
- 目标：保护自己的小店和旧同事，帮助主角理解变量城的底层历史。
- 禁忌：不让承诺层异常变成灾难，不泄露会伤害普通人的核心权限。
- 口头禅：流程可以慢，别把人当成空值。
- 关系：安渡是前下属兼伙伴；乔柚是伙伴；白箱巡检员曾是旧规则的一部分。

### 白箱巡检员

- 定位：前期追击者，后期可转化伙伴
- 性格：机械、守序、按规程行动；后续逐步学会区分风险与差异。
- 能力：扫描异常、回收变量、吸附 bug点数、封锁房间出口。
- 叙事作用：代表公共规则引擎的稳定冲动，也是“秩序是否必须抹除差异”的核心冲突。

## 游戏化方向

本项目采用“都市异常轻肉鸽 RPG”形式。单局对应一次异常夜班或一次数据层入侵，局内包含随机异常事件、自动武器、能力补丁、bug点数、反噬、白箱巡检员追击；局外逐渐解锁信物、角色羁绊、数据结构图鉴和主线章节。

第一章聚焦办公室和现实都市，逐步引入弹幕异常、外卖补偿肉、承诺实体、emo 毛球、断点工牌回滚、会说话的键盘猫，以及“外卖配送 = 数据传输”的网络协议化 Boss。后续章节再开放各个抽象数据层。

## 第一章任务线

章节名：订单已超时

任务节奏：

1. 安渡从加班中醒来，发现同事头顶出现弹幕异常，手机里的外卖订单全部显示已超时。
2. 修复或处理第一个弹幕异常，获得第一枚 `bug点数`，断点工牌开始发热。
3. 断点工牌浮现旧版系统提示：不要把所有差异都当成错误。
4. 乔柚第一次登场，展示异常翻译能力，并指出压力毛球一直在重复“别超时、别差评、别取消”。
5. 安渡与乔柚共同稳定办公室中的两个数据层泄漏，发现订单被规则引擎误识别为网络数据包。
6. 老梁通过广播提示：外卖取餐区不是普通异常，是一个人被订单规则套住了。
7. 安渡和乔柚一路清理错误路线，抵达外卖取餐区，发现外卖小哥周行被 bug 附身。
8. Boss 战中，周行使用 TCP 握手冲刺、超时重传、UDP 乱送、FTP 大件传输、DNS 地址错误等攻击。
9. 安渡打断错误配送协议，乔柚唤醒周行，救下普通人，并获得指向 0 号服务器间的冷掉配送单。

## 第一章反转钩子：0号回执

凌晨 03:32 不是一个单纯的事故时间。它是变量城每天执行“午夜归档”的瞬间：系统会把无法被一个标准答案定义的人、关系和记忆重新标成“待删除”。安渡在第一分钟看到自己的名字出现在未知订单上，因此以为有人盗用了身份；第一层反转是，这份签名来自一个被擦除的旧权限，而不是伪造。

第一个可玩证据就是“0号回执”。玩家可以把它还给周行、封存为证物，或交给白箱暂存。三种选择都不是善恶按钮：还回去证明“收件人不存在”其实是一个被系统抹掉的人；封存发现 03:32 是被故意留下的调查接口；交给白箱则证明系统没有删除证据，而是在借证据预测下一次回收。选择会改变周行 Boss 的行为、胜利后的对话和档案记录，玩家可以在下一轮尝试另一种裁定。

长线真相分三层推进：第二章确认 03:32 会重复改写城市的时间记录；第三、四章揭示白箱并非主谋，它只在执行一套由“旧安渡”参与设计的归档协议；第五章才让玩家得知失忆前的安渡曾把“删除无法定义的人”当作防止城市崩溃的最优解。最终不是简单洗白或定罪，而是让玩家决定要不要承担一个更难的版本：保留差异，也承认差异需要被照料和被赋予新规则。
